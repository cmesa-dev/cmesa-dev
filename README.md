## Software Engineering Portfolio

I build product-oriented software around operational workflows and controlled automation. This portfolio publishes sanitized, executable demos derived from private project work: the code and data shown here are safe to review, run and discuss in a technical interview.

I am looking for software engineering roles on high-impact product teams where ownership, usability, reliability and clear technical decisions matter.

## Featured Work

### [ScanOps - Inventory Operations Terminal](https://github.com/cmesa-dev/barcode-scanner)

An end-to-end product slice for barcode-driven inventory movements: responsive React/TypeScript interface, Node.js REST API, SQLite persistence, transactional stock events, automated tests, Docker packaging and CI verification.

`React` `TypeScript` `Node.js` `SQLite` `REST` `Docker` `GitHub Actions`

<img src="https://raw.githubusercontent.com/cmesa-dev/barcode-scanner/main/docs/scanops-preview.png" alt="ScanOps inventory terminal preview" width="100%"/>

### [KAME - Auditable Operations Workflow](https://github.com/cmesa-dev/kame)

A bounded Python automation workflow that loads synthetic service incidents, prioritises active work, produces a Markdown operations briefing and records a JSON execution trace. Designed to show orchestration and traceability without claiming unimplemented AI behavior.

`Python` `Workflow Orchestration` `Structured Data` `Automated Tests` `GitHub Actions`

### [ONA Studio - Sanitized Frontend Concept](https://github.com/cmesa-dev/onastudio)

A privacy-safe UI concept for a recording studio journey, including responsive layout, room discovery and a session request interaction. It demonstrates visual frontend execution without exposing client assets or private implementation.

`HTML` `CSS` `JavaScript` `Responsive UI` `Accessibility`
